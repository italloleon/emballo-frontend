# Frontend Implementation Guide — Treinos + LLM Generation

> **Target roles:** Admin Panel + Instructor Dashboard
> **Base URL:** `/api/v1`
> **Auth:** Bearer token (Sanctum) in `Authorization` header

---

## 1. Overview

The backend now supports two new areas:

| Area | Who can use | Purpose |
|------|-------------|---------|
| LLM Provider Management | Admin only | Register/manage API keys for OpenAI, Anthropic, or Gemini |
| AI Training Plan Generation | Admin + Instructor | Generate a complete *treino* (training plan) using the academy's LLM |

The existing `TrainingPlan` entity **is** the *treino*. Exercises are stored in the `exercises` catalog; `TrainingExercise` is the pivot (holds sets, reps, rest). Exercises generated by AI are automatically added to the catalog if they don't already exist.

## 2. LLM Provider Management (Admin Panel)

### 2.1 List providers

```
GET /academy/llm-providers
Role: admin
```

Response `200`:
```json
[
  {
    "id": "uuid",
    "academy_id": "uuid",
    "name": "OpenAI Principal",
    "provider": "openai",
    "model": "gpt-4o",
    "active": true,
    "masked_key": "sk-p****...****2aFb",
    "created_at": "2026-05-29T12:00:00Z",
    "updated_at": "2026-05-29T12:00:00Z"
  }
]
```

> **NEVER display** `api_key_encrypted`. The `masked_key` field is the only safe display value.

---

### 2.2 Add a provider

```
POST /academy/llm-providers
Role: admin
```

Body:
```json
{
  "name": "OpenAI Principal",
  "provider": "openai",
  "model": "gpt-4o",
  "api_key": "sk-proj-...",
  "active": true
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | string | ✅ | Display label, max 100 chars |
| `provider` | enum | ✅ | `openai` \| `anthropic` \| `gemini` |
| `model` | string | ✅ | See model list below |
| `api_key` | string | ✅ | Sent once, stored encrypted, never returned in plain text |
| `active` | boolean | ❌ | Default `true` |

**Supported models by provider:**

| provider | Suggested models |
|----------|-----------------|
| `openai` | `gpt-4o`, `gpt-4-turbo`, `gpt-3.5-turbo` |
| `anthropic` | `claude-3-5-sonnet-20241022`, `claude-3-opus-20240229` |
| `gemini` | `gemini-1.5-pro`, `gemini-1.0-pro` |

Response `201`: same schema as list item above.

---

### 2.3 Update a provider

```
PUT /academy/llm-providers/{id}
Role: admin
```

All fields are optional. Omitting `api_key` keeps the existing key unchanged.

```json
{
  "model": "gpt-4-turbo",
  "active": false
}
```

To rotate the API key, include `api_key` with the new value.

---

### 2.4 Delete a provider

```
DELETE /academy/llm-providers/{id}
Role: admin
```

Response `204`.

---

### 2.5 UI recommendations

- Show providers as a settings card, not a full-page table
- The `api_key` input should be `type="password"` or a masked field
- On create success, show the `masked_key` and a banner: _"Chave salva com segurança. Ela não pode ser visualizada novamente."_
- Add a warning before delete if there are active training plans generated by this provider
- Show `active` as a toggle; inactive providers cannot be selected for generation

## 3. AI Training Plan Generation (Instructor + Admin)

### 3.1 Generate a treino

```
POST /training-plans/generate
Role: admin | instructor
Rate limit: 20 requests/hour per user, 100/hour per academy
```

Body:
```json
{
  "student_id": "uuid",
  "goal": "Hipertrofia muscular, foco em membros superiores",
  "context": "Aluno iniciante, 3x por semana, sem histórico de lesões",
  "exercise_count": 6,
  "provider_id": "uuid"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `student_id` | uuid | ✅ | Must belong to same academy |
| `instructor_id` | uuid | Admin only | Required when admin calls this endpoint |
| `goal` | string | ✅ | Free text goal, max 200 chars |
| `context` | string | ❌ | Extra notes for the AI (max 1000 chars), e.g., restrictions, frequency |
| `exercise_count` | integer | ❌ | Default 6, range 3–15 |
| `provider_id` | uuid | ❌ | Uses the first active provider if omitted |

Response `201` — a full `TrainingPlan` with exercises:
```json
{
  "id": "uuid",
  "academy_id": "uuid",
  "instructor_id": "uuid",
  "student_id": "uuid",
  "name": "Treino de Hipertrofia - Membros Superiores",
  "description": "Treino focado no desenvolvimento dos músculos do peitoral, ombros e tríceps...",
  "goal": "hypertrophy",
  "active": false,
  "exercises": [
    {
      "id": "uuid",
      "exercise_id": "uuid",
      "order": 0,
      "sets": 4,
      "reps": "8-12",
      "rest_seconds": 90,
      "notes": "Manter escápulas retraídas durante todo o movimento",
      "properties": {
        "ai_generated": true,
        "provider": "openai"
      },
      "exercise": {
        "id": "uuid",
        "name": "Supino Reto com Barra",
        "description": "...",
        "muscle_groups": ["chest", "triceps"],
        "machine": null
      }
    }
  ],
  "student": { ... },
  "instructor": { ... }
}
```

> Note: `active` is `false` by default on generated plans — the instructor should review and activate manually.

### 3.2 Error responses

| HTTP | Scenario | `message` |
|------|----------|-----------|
| 422 | No active LLM provider configured | `"No active LLM provider configured..."` |
| 422 | Invalid API key or model | `"The LLM provider returned an error..."` |
| 422 | AI returned unparseable JSON | `"The AI returned an invalid response..."` |
| 429 | Rate limit exceeded | Standard throttle response |
| 503 | LLM provider service down | `"...service is temporarily unavailable..."` |

### 3.3 UI flow

#### Generation wizard (3 steps):

**Step 1 — Select student**
- Dropdown or search from the instructor's student list
- Show student name + avatar

**Step 2 — Configure generation**
- `goal` field: text input with quick-select chips (Hipertrofia / Força / Emagrecimento / Resistência / Flexibilidade / Geral)
- `context` field: optional textarea ("restrições, frequência semanal, equipamentos disponíveis...")
- `exercise_count` slider: 3–15, default 6
- If admin: select instructor from dropdown
- "Gerar Treino com IA" button (show spinner + estimated wait: ~15s)

**Step 3 — Review & save**
- Display the generated plan in a review card
- Allow editing: rename plan, change goal, remove exercises, edit sets/reps
- "Ativar e Atribuir" button → calls `PUT /training-plans/{id}` with `{ "active": true }`
- "Descartar" button → calls `DELETE /training-plans/{id}`

## 4. Exercise Catalog (unchanged)

Instructors and admins can manage the exercise catalog independently from training plans.

| Method | Path | Notes |
|--------|------|-------|
| `GET` | `/exercises` | List all catalog exercises |
| `POST` | `/exercises` | Create exercise manually |
| `PUT` | `/exercises/{id}` | Update exercise |
| `DELETE` | `/exercises/{id}` | Delete exercise |

Exercises created by AI generation appear in this catalog and can be reused.

Fields: `name`, `description`, `category`, `muscle_groups` (array), `machine_id` (nullable), `image_url`, `video_url`.

---

## 5. Key security rules (do not break)

- **Never** request or display `api_key_encrypted` from any API response
- The only safe display field is `masked_key` — use it in the settings UI
- The `api_key` input field should use `type="password"` or similar masking
- On a successful key save, display a one-time confirmation message — the raw key cannot be retrieved again
- The `properties.ai_generated` flag on exercises can be used to show a visual indicator (e.g., a small sparkle icon) but should not affect functionality

---

## 6. State machine for generated plans

```
[generated] active=false
     ↓  instructor reviews
[activated] active=true  ← student can now see it at GET /me/training-plans
     ↓  instructor deactivates or creates new plan
[archived] active=false
```

Only `active=true` plans appear on the student's dashboard.
